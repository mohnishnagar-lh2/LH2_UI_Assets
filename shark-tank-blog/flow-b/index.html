Flow B

Pay first, login via email link.

User clicks the CTA, sees a Go Ad-Free landing page, completes payment without creating an account first. After payment, an email with a magic-link is sent — clicking it activates ad-free access on that device and any subsequent ones.

Trade-offs

Pros
Lowest friction at checkout — one less interstitial, no "create an account" step. Higher conversion for impulse subscribers. Identity proven via inbox access without password setup.
Cons
Email round-trip required before access — user has to context-switch to their inbox. Bad email = ghost subscriber: if the user typos their email, they pay but never activate. Refund support load.
When to use
Best when conversion is the primary metric, when readers don't expect or need persistent accounts, or when you want to keep the auth surface minimal (no passwords, no OAuth setup).
Auth pattern
SHA-256 hashed magic-link token in WP transients (30-min TTL). Click → verify token → create user → set HTTP-only cookie → redirect to home with ?loggedIn=1.
